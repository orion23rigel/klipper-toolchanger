# ROADMAP: Per-Tool X/Y Endstop Routing

**Defined:** 2026-06-29
**Project:** Per-Tool X/Y Endstop Routing for Klipper Toolchangers

## Phase 1: Core Module — `tool_axis_endstop.py`

**Goal:** Create the `ToolAxisEndstop` virtual chip class and both config loaders.

**Success Criteria:**

1. `ToolAxisEndstop` class implements full `MCU_endstop` proxy interface (`home_start`, `home_wait`, `query_endstop`, `add_stepper`, `get_steppers`, `setup_pin`)
2. `load_config()` handles unnamed `[tool_axis_endstop]` section with optional `x_default_pin`/`y_default_pin`
3. `load_config_prefix()` handles per-tool `[tool_axis_endstop Tn]` sections with `x_pin`/`y_pin`
4. Virtual chips (`toolchanger_x`, `toolchanger_y`) registered with pins system and printer object system
5. `allow_multi_use_pin` prevents conflicts when probe and endstop share the same physical pin
6. `python3 -m compileall` passes on the new module
7. `has_endstop(tool_number)` public method available for validation

**Tasks:**

- [ ] Create `klipper/extras/tool_axis_endstop.py` with `ToolAxisEndstop` class (~100 lines)
- [ ] Implement `load_config()` — optional unnamed section, default pins
- [ ] Implement `load_config_prefix()` — per-tool sections, parse tool tag from section name
- [ ] Verify CoreXY cross-wiring: `add_stepper()` forwards to all registered endstops
- [ ] Verify shared pin: `allow_multi_use_pin` called before `setup_pin()` for both probe and axis endstop
- [ ] Compile test: `python3 -m compileall klipper/extras/tool_axis_endstop.py`

**Requirements covered:** XEND-01, XEND-02, XEND-03, XEND-04, XEND-05, YEND-01, YEND-02, COMPAT-01, COMPAT-03

## Phase 2: Toolchanger Integration + Validation

**Goal:** Wire the axis endstop routers into the toolchanger lifecycle and add config validation.

**Success Criteria:**

1. `_configure_toolhead_for_tool()` routes X and Y endstops on every tool change
2. `_validate_axis_endstop_coverage()` runs at `klippy:connect` and blocks startup if tools are missing endstop definitions
3. Existing Z probe routing (`tool_probe_endstop.set_active_probe()`) continues to work
4. No regression in existing tool change behavior

**Tasks:**

- [ ] Patch `toolchanger.py` — add X/Y routing in `_configure_toolhead_for_tool()` (~8 lines)
- [ ] Patch `toolchanger.py` — add `_validate_axis_endstop_coverage()` in `_handle_connect()` (~12 lines)
- [ ] Add `has_endstop()` method to `ToolAxisEndstop` class
- [ ] Verify tool change flow: select T0 → X routes to T0's endstop → select T1 → X routes to T1's endstop → select T0 → X routes back to T0's endstop
- [ ] Verify validation: remove one tool's `[tool_axis_endstop Tn]`, confirm startup error lists missing tool

**Requirements covered:** TOOL-01, TOOL-02, VALID-01, VALID-02, VALID-03

## Phase 3: Examples + Documentation

**Goal:** Provide example configs showing the minimal migration path and document the feature.

**Success Criteria:**

1. Example configs in `examples/axis_endstop/` demonstrate the full setup
2. Examples show both X-only and X+Y configurations
3. Migration guide documents the 3 config changes needed
4. Documentation file (`.planning/phase-plan.md` or inline) explains the architecture

**Tasks:**

- [ ] Create `examples/axis_endstop/printer.cfg` — shows `toolchanger_x:x_virtual_endstop`
- [ ] Create `examples/axis_endstop/toolhead_n.cfg` — shows `[tool_axis_endstop Tn]` with `x_pin`, `y_pin`
- [ ] Create `examples/axis_endstop/homing.cfg` — shows homing with per-tool X endstops
- [ ] Verify example configs compile with `compileall`

**Requirements covered:** COMPAT-02 (implicit via examples)
